The Basics of the Web and HTML

The World Wide Web started in the early 1990s and now consists of over 30 billion pages!
The web is a collection of HTML documents. Hyperlinks connect all pages like a web.

HTML (hypertext markup language) is the heart of the web. It is made up of:

  • text content (what we see)
  • markup (what it looks like or how it is arranged)
  • references to other documents (images, videos, etc.)
  • links to other pages (such as this CNN)

Tags are used to instruct a computer what to do.
Most tags consist of an opening and closing tag, but some do not. These exceptions are called Void Tags. One example of a Void tag is the Image tag.

Another interesting point we learned about the use of images is that it is required (as well as courteous) to provide a text description of the image in the event of a broken image or to aid the visually impaired.

While we learned a lot of tidbits regarding href attributes vs.source attributes; inline vs. block elements; and the main components of an HTML document (the Doctype, Head & Body), it is reassuring to know that not all of these details must be committed to memory. Many basic HTML commands are easily found online.

Webpages, Documents, and Structure

A complete HTML document consists of a DOCTYPE declaration, opening and closing html tags that surround the document, the HEAD section that contains the Title and provides linkage to any CSS or Java Script files, opening and closing body tags that surround the content and of course, the content.

HTML and CSS are both languages. HTML cares for structure and CSS cares for style.

HTML has specific rules and syntax. For example, code for block level elements should be on its own line, separate from the content. Additionally, the use of indentation helps to differentiate between code and content and illustrates the parent/child relationship.

DOM (document object model) is a tree like structure that diagrams the HTML commands. Viewing the developer tools in Google Chrome helps to illustrate DOM.

Style-wise, a web page is made up of multiple boxes. All content (text, images, headers, etc.) are their own box contained in an overall box that encompasses the entire page.
The use of class attributes makes organizing and customizing your page more efficient. The class name should be descriptive.

There are many options for writing HTML. We reviewed the benefits of using Scratchpad, Codepen & Sublime. A nice summary can be found here.

Adding style with CSS

With CSS, the most specific rules are applied down in a "cascading" manner. CSS consists of a selector (the element) and a declaration (the style being applied).

Using CSS allows the developer to apply style without using repetitive HTML commands. This allows for increased efficiency in implementing style changes and reduces the opportunity for error. CSS is instantly viewable and allows for responsive design.

One of the key features of CSS is the Inheritance theory, wherein properties are applied to descendants. Inheritance relies on the document tree. The descendants inherit text-related properties, NOT box-related.
Some benefits of Inheritance: faster loading speeds and savings on bandwidth and development costs.

To improve the appearance of the Box layout, the developer adjusts the settings of the margins, borders, padding and context. The overall size of the element = sum of border, padding and content widths.

Boxes can be a set # of pixels or can be set as a % of the of the screen size using FlexBox (very powerful tool in CSS)

To achieve the desired design style, you must Code, Test, Refine. You can test changes using developer tools before adjusting code.

There are HTML and CSS validation tools available to verify that code meets standards.
